Ethereum Gossipsub Mesh Health & Attestation Subnet Imbalances

TL;DR: Ethereum's libp2p mesh shows 2.2x attestation subnet imbalance, relentless mesh maintenance (43M+ graft events in 48h), and healthy validation with only 10 actual failures out of 11M+ rejections.

Key Findings

1. Attestation Subnet Imbalance (2.2x variation)

Attestation subnets show significant traffic variation. Over 6 hours, subnet 28 had 6.18M events while subnet 55 had only 2.83M—a 2.2x difference. This uneven load distribution affects propagation latency; some subnets are naturally "hotter" than others.

2. Relentless Mesh Maintenance

The mesh requires constant graft/prune activity to maintain health:

  • 43,296,300 graft events over 48 hours
  • Hourly range: 52,768 - 987,685 grafts
  • Only 62 actual peer joins/leaves in 24h (topology is stable)

This shows mesh maintenance is continuous background work, not related to peer churn.

3. Healthy Message Validation

Message rejection is dominated by benign causes:

  • 9.7M duplicate messages (normal gossipsub redundancy)
  • 1.6M rejected messages
  • Only 10 actual validation failures
  • 1.3M "validation ignored" (already validated/seen)
  • 352K "validation throttled" (backpressure)

Query: Subnet Distribution

SELECT
    topic_name,
    COUNT() AS events,
    AVG(propagation_slot_start_diff) AS avg_prop_ms
FROM default.libp2p_gossipsub_beacon_attestation
WHERE meta_network_name = 'mainnet'
    AND slot_start_date_time >= now() - INTERVAL 6 HOUR
    AND topic_name LIKE 'beacon_attestation_%'
GROUP BY topic_name
ORDER BY events DESC

Query: Mesh Maintenance

SELECT
    'graft' AS event_type,
    COUNT() AS events,
    countDistinct(peer_id_unique_key) AS unique_peers
FROM default.libp2p_graft
WHERE meta_network_name = 'mainnet'
    AND event_date_time >= now() - INTERVAL 48 HOUR
UNION ALL
SELECT
    'prune' AS event_type,
    COUNT() AS events,
    countDistinct(peer_id_unique_key) AS unique_peers
FROM default.libp2p_prune
WHERE meta_network_name = 'mainnet'
    AND event_date_time >= now() - INTERVAL 48 HOUR

Conclusions

Subnet imbalance is real but bounded. A 2.2x variation is notable but not alarming for a distributed system. It likely reflects natural validator committee distribution.

Mesh maintenance is relentless. The constant graft/prune activity (900K/hour average) shows gossipsub actively maintaining mesh health through local decisions, not just reacting to peer churn.

Network validation health is excellent. With only 10 actual validation failures out of 11M+ rejections, the network is handling load well. Most rejections are benign (duplicates, throttling).
